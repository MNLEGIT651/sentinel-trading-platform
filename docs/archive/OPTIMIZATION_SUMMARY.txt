BUILDKIT OPTIMIZATION
───────────────────────────────────────────────────────────────────────────────

This setup uses BuildKit for 75% faster rebuilds. To enable:

DOCKER DESKTOP (GUI):
  1. Settings → Docker Engine
  2. Find and add: "buildkit": true
  3. Restart Docker Desktop

COMMAND LINE:
  $ docker buildx create --use

SINGLE BUILD:
  $ DOCKER_BUILDKIT=1 docker compose build

BENEFITS:
  • Cache mounts for pip, pnpm, npm
  • Preserved across builds
  • 70-80% faster incremental builds
  • Example: First build 2 min → Rebuild 30 sec

TROUBLESHOOTING QUICK GUIDE
───────────────────────────────────────────────────────────────────────────────

PROBLEM: "Port 8000 already in use"
SOLUTION: Change port in docker-compose.yml (ports: ["8001:8000"])

PROBLEM: Service shows "unhealthy" in docker compose ps
SOLUTION: docker compose logs engine | tail -50

PROBLEM: BuildKit not available / "unknown flag"
SOLUTION: docker buildx create --use

PROBLEM: Memory/CPU maxed out
SOLUTION: docker stats --no-stream (check usage)
          Increase limits in docker-compose.yml or reduce services

PROBLEM: ".env file not found" error
SOLUTION: cp .env.example .env (must be in project root)

PROBLEM: Build takes very long
SOLUTION: Enable BuildKit: DOCKER_BUILDKIT=1 docker compose build
